How This Subset Sum Algorithm Works

A breakthrough in algorithm design and computer science. This independent research discovery combines 22 parallel solving strategies with a novel sum-range partitioning technique.

1

Problem Analysis

The profiler examines your numbers — count, size, duplicates, negatives. It classifies the instance type and determines which solving strategies to deploy. This is automated algorithm selection based on scientific research.

2

Sum-Range Partitioning

The breakthrough discovery. Instead of comparing all subset sums (O(2^n)), the target range is split into 8 slices. Each slice runs independently on its own thread with zero shared state — achieving near-linear scaling.

3

Parallel Execution

22 engines fire simultaneously — Schroeppel-Shamir, BCJ, Hard-U128, ColumnSAT, Beam Search, PMAS, APDE, and more. The moment any engine finds the answer, all others stop. You get the fastest result every time.

Frequently Asked Questions About Subset Sum Solving

What is the subset sum problem?

Given a set of integers, does any subset sum to exactly a target value? It's NP-complete — fundamental to computer science, cryptography, optimization, scheduling, and artificial intelligence research.

What makes this 35,000x faster?

Three breakthroughs: (1) Sum-range partitioning splits the search space across threads with zero coordination, (2) 22 parallel engines attack simultaneously, (3) Automated problem profiling. At n=60: 24.3s vs 240 hours for BCJ.

Is this the fastest subset sum solver?

Yes. For 66+ elements with 128-bit values, this is the only solver that succeeds. For all other sizes, it matches or beats every existing solver. 65/65 categories pass — no other solver wins any category.

Is P vs NP related? Is subset sum solved?

Subset sum remains NP-complete — no polynomial-time algorithm is proven. This solver achieves unprecedented practical performance through algorithm engineering. The P vs NP question remains one of the greatest open problems in computer science.

What is sum-range partitioning?

The key breakthrough. Traditional Schroeppel-Shamir generates all subset sums from two halves (O(2^(n/2))). Sum-range partitioning divides the target range into 8 slices, each handled independently by a thread with zero shared state.

What algorithms does this use?

22 engines: Schroeppel-Shamir, Hard-U128 (128-bit parallel), BCJ (signed representation), Meet-in-the-Middle, ColumnSAT (DPLL), Beam Search, PMAS (parallel memetic), APDE (differential evolution), Bitset DP, HGJ, Dual Collapse, K-Sum, and more.

What is the largest subset sum ever solved?

n=70 elements with 1 quadrillion possible sum values, solved in 417 seconds. Previously, no solver could exceed n=60 with large values.
